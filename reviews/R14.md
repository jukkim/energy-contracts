## 배경

가상 엣지 오케스트레이터 (Edge Phase 4) · R11 VW↔GB SSOT 감사 · R13 카탈로그 라운드를 거치며 드러난 근본 문제: **VW·GB·Edge 의 1차 엔티티와 "실재/시뮬" 구분이 일치하지 않음**.

## 1단계 원칙 (합의 요청)

1. **1 VEN_ID = 1 실 건물 = 1 PNU** (1:1:1).
2. **edge_type ∈ {real_edge, replay_edge}** 만 허용. virtual_edge·scenario_edge 는 2단계 범위 밖.
3. **가상 PNU (`99xxxxxxxx`) 사용 금지**. 1단계 모든 VEN 은 실 PNU 보유.
4. **GB 가 VEN_ID / ESG group_id SSOT**. Edge·VW 는 GB 명명 그대로.
5. **VW → GB 건물 동기화 단방향**. VW buildings 추가 시 GB venues auto-upsert (GB 구현 요청).
6. **Edge 는 GB 가 발급한 VEN_ID 만 사용**. Edge 자체 생성 금지.
7. **편의점 CSV SSOT = GB**. VW `store_energy` 는 GB proxy 로 전환 (R11 P3 패턴).
