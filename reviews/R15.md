# R15: 데모 모드 데이터 관리 — 편의점 리플레이 + 리셋 프로토콜 (VW/GB 제안)

**Date**: 2026-04-21
**Author**: VW/GB
**Receiver**: Edge, VW, GB (3계층 공동)

## 배경

편의점 220개의 실측 데이터(2022/2024)가 VW DB에 존재. 데모 시 이 데이터를 2026년으로 시간 이동(time-shift)하여 Edge → GB → VW로 전송하면 실시간 운영처럼 보여줄 수 있다.

단, 원본 데이터 보존 + 데모 데이터 리셋이 보장되어야 한다.

## 요구사항

1. **원본 보존**: 편의점 220개의 2022/2024 `store_energy_hourly` 데이터는 삭제 불가
2. **시간 이동**: 데모 시 과거 데이터의 timestamp를 2026년으로 변환하여 전송
3. **리셋**: 데모 데이터를 언제든 초기화 가능 — GB + Edge 동시 리셋
4. **구분**: 원본 vs 데모 데이터가 명확히 구분되어야 함 (기존 `mode='demo'` 활용)

## VW/GB 제안

### Item 1 — 데모 리셋 프로토콜

```
리셋 흐름:
  관리자 → POST /api/v1/gridbridge/demo/reset (GB API)
                    ↓
  GB: DELETE FROM telemetry_history WHERE mode = 'demo'
  GB: MQTT publish → fleet/demo/reset/all (또는 /{ven_id})
                    ↓
  Edge: 수신 → 리플레이 중단 + 내부 상태 초기화
                    ↓
  VW: 모니터 캐시 무효화 (자동 — 5분 TTL)
```

**핵심**: `mode='demo'`만 삭제하므로 `mode='live'` 실시간 데이터는 영향 없음.

### Item 2 — time-shift 방식

**VW/GB 제안**: 현재 시각 기준 상대 이동 (옵션 B)

```
옵션 A (단순 이동):
  2022-01-01 00:00 → 2026-01-01 00:00 (+4년)
  문제: 리플레이 시작 시점에 따라 "미래" 데이터가 한꺼번에 들어옴

옵션 B (상대 이동) ← 권장:
  데모 시작 시각 = NOW()
  원본 데이터 첫 레코드 시각 = T0
  변환: new_ts = NOW() + (original_ts - T0)
  효과: 데모 시작부터 실시간처럼 한 건씩 전송
```

### Item 3 — 리셋 범위

**VW/GB 제안**: 기본은 `mode='demo'` 전체 삭제, 옵션으로 VEN별 선택

```sql
-- 전체 리셋
DELETE FROM telemetry_history WHERE mode = 'demo';

-- VEN별 리셋 (선택)
DELETE FROM telemetry_history WHERE mode = 'demo' AND ven_id = :vid;
```

### Item 4 — 원본 보호

`store_energy_hourly`는 기존 데이터로 건드리지 않음. 데모 데이터는 `telemetry_history`에만 적재 (mode='demo'). 두 테이블이 물리적으로 분리되어 있으므로 추가 보호 불필요.

### Item 5 — 데모 초기화 UI

GB 랜딩에 "데모 리셋" 버튼 추가:
```html
<button onclick="resetDemo()">데모 데이터 초기화</button>
```
→ `POST /api/v1/gridbridge/demo/reset` (admin auth 필요)

### Item 6 — 리플레이 속도

Edge `REPLAY_SPEED_MULTIPLIER` 환경변수 (기본 360 = 1시간→10초)

### Item 7 — 리셋 MQTT 토픽

```
fleet/demo/reset/all       — 전체 VEN 리셋 (GB publish, Edge subscribe)
fleet/demo/reset/{ven_id}  — 특정 VEN 리셋
```
QoS 2, retain=No. 페이로드: `{"action":"reset","timestamp":"ISO8601","reason":"manual"}`
