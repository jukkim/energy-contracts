# R16: 3계층 E2E 통합 테스트 계획 — 데모 모드 편의점 5개 (VW/GB 제안)

**Date**: 2026-04-21
**Author**: VW/GB
**Receiver**: Edge, VW, GB (3계층 공동 실행)

## 목적

데모 모드에서 VW → GB → Edge 전체 파이프라인이 정상 동작하는지 검증.
편의점 5개로 소규모 테스트 후 100개로 확대.

## 테스트 시나리오

```
Step 1: 데모 리셋
  POST /api/v1/gridbridge/demo/reset {scope:"all"}
  → GB telemetry_history mode=demo 전부 삭제
  → Edge fleet/demo/reset/all 수신 → ReplayDriver 초기화

Step 2: VW에서 ESG 테스트그룹1 생성
  VW: 새 포트폴리오 "테스트그룹1" + 편의점 5개 할당
  VW → GB: ESG 그룹 자동 생성 + VEN 5개 동기화
  확인: GB /esg/groups → 테스트그룹1 (5 VEN)

Step 3: Edge 연결 + 데이터 전송
  Edge: 테스트그룹1 → 5개 VEN에 가상 생성기 연결
  Edge: 편의점 원본 데이터(2022/2024) → 2026 time-shift (옵션 B)
  Edge: 5초 간격 전송 (REPLAY_SPEED 적절 설정)
  Edge → MQTT → GB telemetry_history (mode=demo, ts=2026)

Step 4: GB 실시간 처리
  4-1. 데이터 수집: telemetry_history 적재 확인 (5 VEN × N건)
  4-2. 실시간 모니터링: GB dashboard.html?group=테스트그룹1 → 5 VEN 카드
  4-3. 168→24 예측: 168시간 데이터 축적 후 Korean_BB 예측 트리거
  4-4. 이상치 탐지: forecast residual 기반 anomaly score 계산

Step 5: 결과 비교
  5-1. 데모 데이터 vs 원본 데이터 (2022/2024) 동일성 검증
  5-2. EUI 계산 결과 비교 (demo telemetry vs store_energy_hourly)
  5-3. 이상치 비율 비교
```

## 편의점 5개 선택 기준

VW `store_energy_hourly`에서 데이터 품질이 좋은 5개 선택:
- 데이터 연속성 높은 것 (gap 적은 것)
- EUI 분포 다양 (고효율 1 + 중간 2 + 저효율 2)
- 이상치 존재하는 것 1개 이상 (탐지 테스트)

## 성공 기준

| 항목 | 기준 |
|------|------|
| 데이터 적재 | 5 VEN × 168시간 이상 demo 데이터 축적 |
| 모니터링 | GB dashboard에 5 VEN 카드 실시간 표시 |
| VW 모니터 | monitor.html에 5 VEN (데모) 표시 |
| 168→24 예측 | 168시간 축적 후 24시간 예측 생성 (CVRMSE < 15%) |
| 이상치 | 원본 데이터의 이상치와 동일한 패턴 재현 |
| 데이터 비교 | demo ts 변환 후 원본과 값(power_kw) 100% 일치 |
| 리셋 | 데모 리셋 후 demo=0건, live 영향 없음 |
